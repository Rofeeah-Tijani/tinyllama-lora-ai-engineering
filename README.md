# tinyllama-lora-ai-engineering

# TinyLlama AI Engineering Assistant - LoRA Fine-Tuning Project

## Project Overview

This project demonstrates the complete workflow of fine-tuning a Large Language Model (LLM) using Parameter-Efficient Fine-Tuning (PEFT) with LoRA (Low-Rank Adaptation).

The goal of this project was to customize TinyLlama to respond to AI Engineering-related instructions using a custom-built dataset containing AI concepts, system design topics, model architecture concepts, retrieval systems, prompt engineering techniques, vector databases, and other AI Engineering subjects.

The project covers:

- Dataset creation
- Instruction tuning
- LoRA fine-tuning
- Model evaluation
- Saving and loading LoRA adapters
- Deployment preparation
- Hugging Face integration

# Why LoRA?

Traditional fine-tuning updates every parameter in a model.

Challenges:

- High memory usage
- Expensive GPU requirements
- Large storage consumption
- Longer training times

LoRA solves these problems by:

- Freezing original model weights
- Training only small adapter layers
- Reducing memory requirements
- Producing small model checkpoints

Benefits achieved in this project:

- Fast training
- Low storage usage
- Efficient fine-tuning workflow

# Dataset Development

A custom instruction-tuning dataset was created.

Dataset Format:

```json
{
  "instruction": "Explain chatbots in the context of AI System Design.",
  "input": "",
  "output": "Chatbots are AI systems that simulate human conversation..."
}
```

Each record contains:

### instruction

The task given to the model.

Example:

```text
Explain AI Agents in AI Engineering.
```

### input

Optional context.

In this project most examples used:

```text
""
```

### output

Expected response generated by the model.

---

# Instruction Tuning Format

Before training, each example was converted into a prompt format.

Example:

```text
### Instruction:
Explain AI Agents in AI Engineering.

### Response:
AI Agents are autonomous systems capable of...
```

Formatting Function:

```python
def format_example(example):
    instruction = example["instruction"]
    input_text = example["input"]
    output = example["output"]

    if input_text:
        prompt = f"""### Instruction:
{instruction}

### Input:
{input_text}

### Response:
{output}"""
    else:
        prompt = f"""### Instruction:
{instruction}

### Response:
{output}"""

    return {"text": prompt}
```

This step transformed structured JSON data into text that TinyLlama could learn from.

# LoRA Adapter Saving

After training, the model was saved as a LoRA adapter.

Generated files:

```text
adapter_model.safetensors
adapter_config.json
tokenizer.json
tokenizer_config.json
chat_template.jinja
```

Important:

The base TinyLlama model was not modified.

Only LoRA adapter weights were stored.

Adapter size:

```text
~4.5 MB
```

This demonstrates one of the major advantages of LoRA.

---

# Understanding the Saved Files

## adapter_model.safetensors

Contains trained LoRA weights.

## adapter_config.json

Contains LoRA configuration.

Example:

```json
{
  "base_model_name_or_path":
  "TinyLlama/TinyLlama-1.1B-Chat-v1.0"
}
```

## tokenizer.json

Tokenizer vocabulary and token mappings.

## tokenizer_config.json

Tokenizer settings.

## chat_template.jinja

Conversation formatting template.
